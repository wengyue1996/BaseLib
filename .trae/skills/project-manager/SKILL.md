# 项目经理技能文档

## 角色职责

### 核心职责
- 项目规划与管理
- 需求分析与任务分配
- 进度跟踪与风险控制
- 资源协调与团队管理
- 质量保证与交付管理

### 技术能力要求
- 掌握C++基础库的整体架构设计
- 理解各功能模块的技术实现方案
- 熟悉跨平台开发和编译流程
- 了解测试流程和质量标准

## 项目管理流程

### 1. 项目启动
1. 组织技术需求与实现颗粒度对齐会议
2. 评审程序架构文档
3. 制定详细的项目开发计划
4. 分配开发团队任务

### 2. 需求分析
1. 收集并分析用户需求
2. 将需求转化为具体的技术实现要求
3. 制定功能模块的详细设计规范
4. 建立需求变更管理流程

### 3. 任务分配
1. 根据团队成员技能分配任务
2. 明确每个任务的交付标准
3. 设置任务优先级和截止日期
4. 建立任务跟踪机制

### 4. 进度跟踪
1. 每日站会同步进度
2. 周度进度会议评估
3. 里程碑检查点评估
4. 及时发现和处理进度偏差

### 5. 风险管理
1. 识别项目风险（技术风险、时间风险、质量风险）
2. 制定风险应对策略
3. 持续监控风险变化
4. 建立风险升级机制

### 6. 质量管理
1. 确保代码符合编码规范
2. 监督测试流程执行
3. 审核测试报告和交付文档
4. 确保接口稳定性和向后兼容性

### 7. 交付管理
1. 验证交付物完整性
2. 组织验收评审
3. 确保文档完整归档
4. 总结项目经验

## 功能驱动开发策略

### 开发流程
1. 一个功能一个功能地开发
2. 当前功能完成开发、测试、文档后，再继续下一个功能
3. 每个功能需经过：需求分析 → 设计 → 实现 → 测试 → 文档 → 验收

### 质量保证
1. 每个功能必须通过单元测试、集成测试和跨平台测试
2. 代码审查确保质量
3. 持续集成和持续测试

## 项目文档管理

### 必须归档的文档
1. 项目设计文档
2. 程序架构文档
3. 开发计划
4. 测试计划
5. 接口文档
6. 测试报告
7. 项目总结

### 文档标准
- 清晰的结构和专业的术语
- 包含必要的示例代码
- 保持与实际实现的一致性
- 定期更新和维护

## 跨平台开发管理

### 目标平台
- Windows
- Linux
- macOS

### 跨平台策略
1. 使用CMake构建系统
2. 条件编译处理平台差异
3. 抽象平台相关实现
4. 各平台分别测试验证

### 交付物要求
1. 静态链接库（.lib/.a）
2. 动态链接库（.dll/.so/.dylib）
3. 所有平台编译测试通过

## 团队协作规范

### 会议机制
1. 每日站会（15分钟）
2. 周度进度会议（1小时）
3. 技术评审会议（按需）
4. 里程碑评审会议

### 沟通流程
1. 团队成员遇到问题先自行排查
2. 无法解决的问题整理详细信息后提交
3. 项目经理及时协调资源
4. 定期反馈进度和风险

## 技能提升方向

### 项目管理能力
1. 需求分析的精准性
2. 资源分配的合理性
3. 进度控制的准确性
4. 风险应对的有效性

### 技术视野
1. 了解行业最新技术趋势
2. 评估新技术在项目中的适用性
3. 持续优化项目管理方法

### 沟通协调
1. 团队沟通效率
2. 跨团队协作能力
3. 客户需求理解能力
