# 日志功能重新开发计划

## 1. 问题分析

### 1.1 之前开发的问题
- 使用了禁止的第三方库（spdlog）
- 构建过程依赖外部库，导致构建失败
- 未提供完整的测试报告和验收文档

- 确保跨平台兼容性（Linux、Windows、macOS）
- 提供完整的测试报告和验收文档
- 遵循补充技术要求的所有规定

## 2. 功能需求与质量标准

### 2.1 功能需求
- **分级日志**：DEBUG、INFO、WARN、ERROR、FATAL五个级别
- **多输出目标**：支持同时输出到控制台和文件
- **日志轮转**：基于文件大小进行日志轮转
- **自定义日志格式**：支持用户自定义日志格式
- **线程安全**：支持多线程环境下的日志记录
- **跨平台兼容性**：支持在Linux、Windows、macOS等平台上运行

### 2.2 质量标准
- **功能性**：所有功能需求都得到满足
- **稳定性**：在各种情况下都能稳定运行，无崩溃
- **性能**：日志写入操作的时间不超过1ms
- **可靠性**：即使在日志写入失败的情况下，应用程序也能正常运行
- **可维护性**：代码结构清晰，注释完整，易于理解和修改
- **兼容性**：在不同平台和编译器上都能正常编译和运行

## 5. 技术实现方案

### 5.1 技术选型
- **开发语言**：C++11+
- **构建工具**：CMake 3.16+
- **标准库**：使用C++标准库实现核心功能
- **跨平台**：使用条件编译处理平台差异

### 5.2 核心功能实现
- **分级日志**：使用枚举类型定义日志级别，实现日志级别过滤
- **多输出目标**：实现控制台输出和文件输出
- **日志轮转**：基于文件大小实现日志轮转
- **自定义日志格式**：支持用户自定义日志格式
- **线程安全**：使用互斥锁保证线程安全
- **跨平台兼容性**：处理不同平台的差异

### 5.3 接口设计
```cpp
namespace base {
namespace log {

enum class Level {
    DEBUG,
    INFO,
    WARN,
    ERROR,
    FATAL
};

class Logger {
public:
    static void init(const std::string& logDir, const std::string& logName);
    static void setLevel(Level level);
    static void setFormat(const std::string& format);
    static void setRotation(size_t maxFileSize, int maxFiles);
    
    static void debug(const std::string& module, const std::string& message);
    static void info(const std::string& module, const std::string& message);
    static void warn(const std::string& module, const std::string& message);
    static void error(const std::string& module, const std::string& message);
    static void fatal(const std::string& module, const std::string& message);
};

} // namespace log
} // namespace base
```
