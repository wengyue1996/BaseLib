## 2. 核心功能设计

### 2.1 分级日志

#### 2.1.1 设计思路
- 使用枚举类型定义日志级别
- 基于spdlog的日志级别实现
- 提供设置日志级别的接口

#### 2.1.2 实现细节
- 定义Level枚举，包含DEBUG、INFO、WARN、ERROR、FATAL五个级别
- 提供setLevel方法，用于设置日志级别
- 在内部将自定义的Level映射到spdlog的级别

### 2.2 多输出目标

#### 2.2.1 设计思路
- 使用spdlog的多sink机制
- 支持同时输出到控制台和文件

#### 2.2.2 实现细节
- 创建控制台sink和文件sink
- 将两个sink添加到同一个logger中
- 分别设置不同的日志格式

### 2.3 日志轮转

#### 2.3.1 设计思路
- 使用spdlog的rotating_file_sink
- 基于文件大小进行日志轮转

#### 2.3.2 实现细节
- 创建rotating_file_sink，设置最大文件大小和最大文件数量
- 当日志文件达到最大大小时，自动创建新的日志文件
- 最多保留指定数量的日志文件

### 2.4 自定义日志格式

#### 2.4.1 设计思路
- 使用spdlog的pattern设置
- 支持用户自定义日志格式

#### 2.4.2 实现细节
- 提供setFormat方法，用于设置日志格式
- 支持常见的格式化占位符，如时间、级别、文件、行号等

### 2.5 线程安全

#### 2.5.1 设计思路
- 使用spdlog的线程安全sink
- 加锁保护共享资源

#### 2.5.2 实现细节
- 使用std::mutex保护logger的访问
- 使用spdlog的mt（多线程）版本的sink

### 2.6 跨平台兼容性

#### 2.6.1 设计思路
- 使用spdlog的跨平台实现
- 处理平台特定的差异

#### 2.6.2 实现细节
- 依赖spdlog的跨平台特性
- 确保在不同平台上的编译和运行

## 3. 接口设计

### 3.1 公共接口

```cpp
namespace base {
namespace log {

enum class Level {
    DEBUG,
    INFO,
    WARN,
    ERROR,
    FATAL
};

class Logger {
public:
    static void init(const std::string& logDir, const std::string& logName);
    static void setLevel(Level level);
    static void setFormat(const std::string& format);
    static void setRotation(size_t maxFileSize, int maxFiles);
    
    static void debug(const std::string& module, const std::string& message);
    static void info(const std::string& module, const std::string& message);
    static void warn(const std::string& module, const std::string& message);
    static void error(const std::string& module, const std::string& message);
    static void fatal(const std::string& module, const std::string& message);
};

} // namespace log
} // namespace base
```

### 3.2 接口说明

| 接口 | 功能 | 参数 | 返回值 |
|------|------|------|--------|
| init | 初始化日志系统 | logDir: 日志目录<br>logName: 日志文件名 | 无 |
| setLevel | 设置日志级别 | level: 日志级别 | 无 |
| setFormat | 设置日志格式 | format: 日志格式字符串 | 无 |
| setRotation | 设置日志轮转 | maxFileSize: 最大文件大小<br>maxFiles: 最大文件数量 | 无 |
| debug | 输出DEBUG级别的日志 | module: 模块名<br>message: 日志消息 | 无 |
| info | 输出INFO级别的日志 | module: 模块名<br>message: 日志消息 | 无 |
| warn | 输出WARN级别的日志 | module: 模块名<br>message: 日志消息 | 无 |
| error | 输出ERROR级别的日志 | module: 模块名<br>message: 日志消息 | 无 |
| fatal | 输出FATAL级别的日志 | module: 模块名<br>message: 日志消息 | 无 |

## 4. 实现细节

### 4.1 初始化过程

1. 创建控制台sink，设置彩色输出和格式
2. 创建文件sink，设置轮转参数和格式
3. 创建logger，添加两个sink
4. 设置默认日志级别和刷新策略

### 4.2 日志输出过程

1. 检查logger是否初始化
2. 加锁保护共享资源
3. 根据日志级别调用相应的spdlog方法
4. 解锁共享资源

### 4.3 日志轮转过程

1. 当日志文件达到最大大小时，spdlog会自动创建新的日志文件
2. 最多保留指定数量的日志文件，超出后会删除最旧的文件

### 4.4 线程安全实现

1. 使用std::mutex保护logger的访问
2. 使用spdlog的mt（多线程）版本的sink
3. 确保在多线程环境下的安全访问

## 6. 性能优化

### 6.1 优化策略

- 使用spdlog的异步模式，减少对主线程的影响
- 批量写入日志，减少系统调用
- 合理设置日志级别，避免不必要的日志输出
- 使用缓冲区，减少磁盘IO操作

### 6.2 性能指标

- 日志写入时间：≤1ms
- 内存使用：≤1MB
- CPU使用率：≤1%

## 7. 错误处理

### 7.1 错误类型

- 初始化错误：日志目录不存在、权限不足等
- 写入错误：磁盘空间不足、文件权限不足等
- 配置错误：无效的日志级别、格式等

### 7.2 错误处理策略

- 初始化错误：抛出异常，提示用户
- 写入错误：记录错误信息，继续运行
- 配置错误：使用默认值，继续运行

## 8. 代码规范

### 8.1 命名规范

- 类名：PascalCase（如 `Logger`）
- 函数名：camelCase（如 `init()`）
- 变量名：camelCase（如 `logDir`）
- 常量名：ALL_CAPS（如 `MAX_FILE_SIZE`）
- 枚举名：PascalCase（如 `Level`）

### 8.2 代码风格

- 缩进：4个空格
- 括号：K&R风格
- 行宽：不超过80字符
- 注释：关键代码必须有注释

### 8.3 异常处理

- 使用异常处理错误情况
- 异常信息要清晰、准确
- 避免在日志系统中使用日志记录错误（避免递归）

## 9. 部署与集成

### 9.1 构建系统

- 使用CMake构建系统
- 支持静态库和动态库的构建

### 9.2 依赖管理

- 依赖spdlog库
